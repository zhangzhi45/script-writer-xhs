原文：

如果你还因为安装问题，或者安全问题，到现在还没用上小龙虾的话，那腾讯出的这个 WorkBuddy 适合 99% 的人去使用。

我已经替大家重度使用两周了，说实话，我最近小龙虾都打开得少了，基本上日常办公的场景我都在用它。

今天我就来对比一下，为什么我觉得它比小龙虾更适合我们普通用户。

首先是安装。

小龙虾的安装大家应该都有体会，要么用脚本、要么用 Docker、要么靠云端部署，总之不折腾一番是用不上的。WorkBuddy 就简单太多了——直接去官网下载，安装，完事了。不用配环境，不用搞命令行，而且完全免费。

第二个是安全问题。

这个其实很多人在意。小龙虾装到你电脑上以后，它拥有的是整台电脑的权限，我就担心它乱删我的文件，所以一直没在主力机上部署。WorkBuddy 不一样，它不会随便动你电脑上的东西，要删除文件的话会先弹出确认，让你决定要不要删。对普通用户来说放心很多。

第三个，也是很多人关心的：能力上有没有差距？

说实话，小龙虾有的能力，WorkBuddy 基本上也都有。我来一个一个说。

**多 Agent 协作。** 你跟它说"帮我组建团队"，它会自动调度多个 Agent 来配合完成任务。我就用它建了一个产研团队——一个产品经理、一个架构师、还有一个项目经理来统筹。我还给他们分别设定了不同的角色，创建了人物小传，这样每个 Agent 就会更加立体，做出来的东西也更贴合实际需求。

**Skill 系统。** 这个我要重点说一下——WorkBuddy 的 Skill 和小龙虾的 Skill 是完全兼容的。什么意思呢？就是你之前在小龙虾上用的那些 Skill，直接搬过来就能用，不需要做任何修改。

这一点挺重要的，因为很多人在小龙虾上已经积累了不少好用的 Skill，换个工具如果不兼容，等于之前的东西都白做了。WorkBuddy 完全没有这个问题。

**MCP 工具**也一样，Claude 生态里那些工具，WorkBuddy 都能用。但你不需要自己去配置，它已经帮你处理好了。

我来举两个我自己实际在用的场景。

**第一个：数据分析。**

为了演示，我先让它生成一个销售数据的 Excel 表。好，生成了，我们看一下——数据还是挺丰富的。

然后我跟它说：帮我分析一下这个销售数据的趋势，生成图表，最后输出一份完整的分析报告。

它生成的报告，真的比我们自己整理的还要好。整个过程也就五分钟。

**第二个：日常办公自动化。**

我日常工作开完会都会录音，录完以后我会让 WorkBuddy 去读取录音内容，帮我转成文字，写成会议纪要，然后发送给相关同事的邮箱。整个过程也就几分钟就搞定了。

好了，如果你也想用 AI Agent 来提升效率，但又不想折腾安装配置，也担心安全问题，WorkBuddy 真的是一个很好的选择。

建议先从简单的任务开始，比如整理个文档、分析个表格，慢慢你就会发现它能帮你做的事情比你想象的多。
