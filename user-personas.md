# User Personas — 目标用户画像

## 3类核心用户

### 用户A：AI新手（占比40%）

**基本信息**
- 年龄：25-35岁
- 职业：白领、学生、自由职业者
- 痛点：想用AI但不知道从何开始，怕被割韭菜

**需求层次**
1. **基础需求**：了解AI能做什么，不会被骗
2. **功能需求**：找到简单好用的AI工具
3. **情感需求**：感觉自己跟上了时代，不焦虑
4. **自我实现**：用AI提升工作效率，获得成就感

**触达策略**
- 内容方向：入门教程、避坑指南、工具推荐
- 关键词："小白也能懂"、"零基础"、"免费"
- 痛点切入："很多人不知道..."、"我刚入门时也..."

**典型场景**
- 看到AI新闻但看不懂
- 想尝试ChatGPT但不知道怎么用
- 担心付费工具不值得

---

### 用户B：效率追求者（占比35%）

**基本信息**
- 年龄：28-40岁
- 职业：产品经理、运营、设计师、程序员
- 痛点：工作太忙，希望用AI提效，但工具太多选不过来

**需求层次**
1. **基础需求**：找到适合自己工作场景的AI工具
2. **功能需求**：学会具体的使用方法和工作流
3. **情感需求**：成为同事眼中的"效率达人"
4. **自我实现**：用AI创造更大价值，获得晋升/收入提升

**触达策略**
- 内容方向：工作流搭建、工具组合、实战案例
- 关键词："效率翻倍"、"自动化"、"工作流"
- 痛点切入："每天重复的工作..."、"这个方法帮我节省了..."

**典型场景**
- 每天处理大量重复性工作
- 想自动化某些流程但不知道怎么做
- 需要向老板/客户展示AI带来的效率提升

---

### 用户C：技术爱好者（占比25%）

**基本信息**
- 年龄：22-35岁
- 职业：程序员、技术产品经理、独立开发者
- 痛点：想了解AI技术原理和最新趋势，但中文资料少

**需求层次**
1. **基础需求**：了解AI技术原理和发展趋势
2. **功能需求**：掌握高级用法，能二次开发
3. **情感需求**：在技术社区有影响力，被认可
4. **自我实现**：用AI做出有影响力的产品/项目

**触达策略**
- 内容方向：技术解读、源码分析、趋势预测
- 关键词："底层原理"、"源码"、"架构"
- 痛点切入："很多人没注意到的细节..."、"从源码来看..."

**典型场景**
- 关注AI论文和技术博客
- 想自己开发AI应用或插件
- 在技术社区分享和讨论

## 4层需求模型

```
┌─────────────────────────────────────┐
│  第4层：自我实现                      │
│  用AI创造价值，获得成就感和影响力       │
├─────────────────────────────────────┤
│  第3层：情感需求                      │
│  不焦虑、被认可、有归属感              │
├─────────────────────────────────────┤
│  第2层：功能需求                      │
│  学会使用工具，解决具体问题            │
├─────────────────────────────────────┤
│  第1层：基础需求                      │
│  了解AI，不被时代抛弃                 │
└─────────────────────────────────────┘
```

**内容策略**：
- 新手内容：重点满足1-2层需求
- 进阶内容：重点满足2-3层需求
- 深度内容：重点满足3-4层需求

---

## 内容覆盖原则

每篇内容尽量覆盖≥2类用户：

| 内容类型 | 主要用户 | 次要用户 | 覆盖策略 |
|---------|---------|---------|---------|
| 入门教程 | 新手 | 效率追求者 | 新手能懂，效率追求者能学到新技巧 |
| 工具推荐 | 效率追求者 | 新手 | 效率追求者直接可用，新手了解可能性 |
| 技术解读 | 技术爱好者 | 效率追求者 | 技术爱好者看深度，效率追求者看应用场景 |
| 实战案例 | 效率追求者 | 技术爱好者 | 效率追求者复制使用，技术爱好者学习思路 |

## 用户语言习惯

### 新手常用语
- "这个难吗？"
- "需要花钱吗？"
- "我零基础能学会吗？"
- "会不会很复杂？"

### 效率追求者常用语
- "能节省多少时间？"
- "和我的工作流能结合吗？"
- "有没有现成的模板？"
- "稳定性怎么样？"

### 技术爱好者常用语
- "底层是怎么实现的？"
- "和XXX相比有什么优势？"
- "支持自定义吗？"
- "有API吗？"

---

## 内容调性匹配

| 用户类型 | 语气 | 深度 | 节奏 |
|---------|------|------|------|
| 新手 | 鼓励、耐心 | 浅入浅出 | 慢，多解释 |
| 效率追求者 | 实用、直接 | 深入浅出 | 中等，重方法 |
| 技术爱好者 | 专业、客观 | 深入深出 | 快，重逻辑 |
