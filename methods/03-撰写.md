# Method 03 — 正文撰写

## 适用场景

- 大纲已确认
- 需要撰写完整文稿
- 必须参考语气样本文稿

---

## 执行流程

### Step 0：读取爆款索引

**撰写前，必须先读取 `Vault/爆款脚本索引.md`**

分析历史爆款的表达特点，借鉴已验证的开头结构、内容节奏和结尾方式。

### Step 1：检查文稿大纲

**如果用户提供了文稿大纲，必须严格按照大纲结构撰写文稿。**

**借鉴思路**：
- 工具类内容必须有具体场景演示
- 用"我已经重度使用两周"增加可信度
- 结尾要有明确的互动问题

### Step 2：读取语气样本文稿

**必须执行**：读取 `references/tone-samples/` 中的所有样本文稿，深入理解用户的真实语气风格。

**样本文稿特点**：
- 开场自然，常用"今天跟大家分享..."
- 过渡平滑，善用"什么意思呢？"
- 结尾温和，常用"好了今天就分享到这里"
- 整体像朋友间平静聊天，不夸张不套路

### Step 3：撰写开头（3-5秒抓住注意力）

**开头原理：必须回应三个潜意识问题**

开头的前3秒决定用户是否继续观看，必须同时回应：

1. **跟我有什么关系？**（用具体痛点触发共鸣）
2. **为什么听你的？**（建立可信度，如"重度使用两周"）
3. **我能得到什么？**（清晰点出核心价值）

**开头要有情绪**：
- ❌ 陈述事实："AI现在有自己的社交网络了"
- ✅ 传递情绪："我刚逛了一下全都是AI在聊天的论坛，惊到我了"

**优秀案例参考（来自爆款索引）**：

**CoPAW式开头**（痛点前置）：
> "很多人想装OpenClaw但装不上，要么用脚本、要么用Docker..."

**WorkBuddy式开头**（痛点+对比）：
> "如果你还因为安装问题，或者安全问题，到现在还没用上小龙虾的话..."

**开头句式来源（按优先级）**：
1. **爆款索引参考** - 优先使用验证过的爆款开头结构
2. **素材库推荐** - 使用 `Vault/01-灵感与素材库/05-核心概念库/03-开场句式模板/`
3. **常用开场库** - about-me.md 中的常用开场

**开头公式（工具推荐类）**：
1. **痛点前置型**："很多人想【做某事】但【遇到痛点】..."
2. **对比型**："如果你还因为【痛点A】或【痛点B】，还没用上【工具】..."

**开场句式模板（来自素材库）**：
- 震惊型："你知道吗，[反常识事实]"
- 痛点型："还在用[旧方法]？[痛点描述]"
- 悬念型："今天这个[东西]，可能会改变你[领域]的认知"
- 共鸣型："如果你也[共同经历]，这条视频一定要看完"
- 利益型："今天给你[数量]个[东西]，直接[达成的效果]"

**about-me.md 常用开场**：
- "很多人[做完某件事]之后，都不知道下一步该干什么..."
- "给大家分享 N 个我自己每天都在高频用的..."
- "我觉得这是目前为止..."
- "给大家分享一个我最近..."
- "这应该是我见过最好的...了"
- "我也是今天发现的，忍不住跟大家分享"

### Step 4：撰写主体内容

**滑梯理论**：环环相扣，层层递进，但不引发焦虑

**信息密度**：
- **实操前绝对不废话**：进入核心内容或实操演示前的过渡句不能超过两句话。
- 如果一句话去掉后不影响表达，就去掉
- 每句话都应该有存在的理由
- 内容能产生行为或认知上的改变

**句式特点**：
- 短句为主，多用"其实"、"呢"、"吧"等轻微语气词
- 人称灵活，根据内容使用"我"或"我们"
- 保留真人停顿感，不能太完美、太平滑

**过渡表达**：
- "什么意思呢？"
- "我把它总结为一个公式：..."
- "就用...去类比下"
- "我举个例子"

### Step 5：撰写结尾

**参考爆款索引的结尾方式**：

**CoPAW式结尾**（行动号召）：
> "好了，如果你也想用AI Agent来提升效率...建议先从简单的任务开始..."

**WorkBuddy式结尾**（互动引导）：
> "你平时工作中有什么重复性的任务想交给AI来做？欢迎在评论区聊聊。"

**结尾公式（工具推荐类）**：
1. **行动号召型**：总结价值 + 使用建议 + 鼓励尝试
2. **互动引导型**：总结价值 + 开放式问题 + 邀请评论

**选择以下方式之一**：
- **开放式问题**："你有没有类似的经历？"、"你会怎么选？"
- **行动建议**："如果觉得有用，可以点赞收藏一下"
- **引导互动**："如果大家感兴趣的话，可以在评论区留言"

**常用收尾**：
- "这样一来...再也不用担心..."
- "如果大家感兴趣的话..."
- "建议你也可以去试试这个方法"
- "好了今天就分享到这里"

## 撰写检查清单

| 检查项 | 合格标准 |
|-------|---------|
| **🚨 2秒退出率（最优先）** | 开头无"前情提要"；教程类建议前2秒先放操作成果画面 |
| **历史语气一致性** | 与之前写的文稿语气保持一致，参考 `references/tone-samples/` 中的真实表达 |
| 语气风格 | 整体像朋友聊天，不夸张不套路 |
| 开头情绪 | 用"我刚发现/忍不住"等真实表达，或直接抛出痛点 |
| 3个问题 | 回应"跟我有什么关系/为什么听你的/能得到什么" |
| 禁止词汇 | 无"谁懂啊/救命/绝了/太牛了"等 |
| 滑梯理论 | 环环相扣，层层递进 |
| 信息密度 | 每句话都有存在理由，无废话 |
| 言之有物 | 能产生行为或认知改变 |
| 句式风格 | 短句为主，用"其实/呢/吧"等语气词 |
| 结尾互动 | 有开放式问题/行动建议/引导互动 |
| AI腔检查 | 不用"作为AI模型"等解释性文字 |
| 真实感 | 保留真人停顿感，不套路不夸张 |
| 人称一致 | "我"或"我们"使用统一 |

---

## 长度控制

**核心原则**：该长则长，该短则短，一切服务于内容本身。

- 不硬性限制字数或时长
- 简单的工具推荐不要注水拉长
- 复杂的深度拆解不要为了控制篇幅而砍掉关键信息
- 唯一的硬约束：**没有废话**

---

## 输出格式

### 默认输出：纯文稿

直接输出完整文案，不要任何说明、分析、标注。

### 可选输出：文稿 + 分镜

如用户说"加分镜"或"需要分镜"，则在每段后添加：

```
【画面提示】此处展示 XXX 截图/演示 XXX 操作
```

---

## 撰写完成后

1. 自检：对照上面的检查清单逐项检查
2. 输出：输出完整文稿
3. 等待用户反馈，或进入 Method 05（评分）
