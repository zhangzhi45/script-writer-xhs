# Method 03 — 正文撰写

## 适用场景

- 大纲已确认
- 需要撰写完整文稿
- 必须参考语气样本文稿

---

## 执行流程

### Step 1：读取语气样本文稿

**必须执行**：读取 `references/tone-samples/` 中的所有样本文稿，深入理解用户的真实语气风格。

**样本文稿特点**：
- 开场自然，常用"今天跟大家分享..."
- 过渡平滑，善用"什么意思呢？"
- 结尾温和，常用"好了今天就分享到这里"
- 整体像朋友间平静聊天，不夸张不套路

### Step 2：撰写开头（3-5秒抓住注意力）

**黄金前3秒原则：**
第一句话绝不能是平淡的铺垫（如"今天给大家分享几个用法"）。必须立刻前置展示最牛的成果、最抓人的场景或直击最痛的痛点（如"看我怎么用AI自动提取这篇文章的精华"），否则用户会立刻划走。

**🚫 绝对禁止"前情提要"式开场（会直接导致2秒退出率飙升）：**
- ❌ "上一期教大家安装了 XXX，接下来..."
- ❌ "今天继续讲上次的内容..."
- ❌ "我之前发过一个帖子，那里提到..."

原因：小红书每条视频对路人来说都是独立存在的。一开口"前情提要"，路人立刻判断"这不是给我看的"，直接划走，导致2秒退出率飙升，算法断流。如需关联前作，放在视频末尾或评论区置顶即可。

**🎯 先展示成果，再开口（针对教程/工具类）：**
开口之前的前2秒，建议先放最震撼的操作成果屏录（如飞书自动写入数据、AI自动输出报表），让视觉冲击替代语言留住用户，再切到口播正文。

开头必须回应三个潜意识问题：
1. **跟我有什么关系？**（用具体痛点触发共鸣）
2. **为什么听你的？**（建立可信度）
3. **我能得到什么？**（清晰点出核心价值）

**开头要有情绪**：
- ❌ 陈述事实："AI现在有自己的社交网络了"
- ✅ 传递情绪："我刚逛了一下全都是AI在聊天的论坛，惊到我了"

**常用开场**：
- "很多人[做完某件事]之后，都不知道下一步该干什么..."
- "给大家分享 N 个我自己每天都在高频用的..."
- "我觉得这是目前为止..."
- "给大家分享一个我最近..."
- "这应该是我见过最好的...了"
- "我也是今天发现的，忍不住跟大家分享"

### Step 3：撰写主体内容

**滑梯理论**：环环相扣，层层递进，但不引发焦虑

**信息密度**：
- **实操前绝对不废话**：进入核心内容或实操演示前的过渡句不能超过两句话。
- 如果一句话去掉后不影响表达，就去掉
- 每句话都应该有存在的理由
- 内容能产生行为或认知上的改变

**句式特点**：
- 短句为主，多用"其实"、"呢"、"吧"等轻微语气词
- 人称灵活，根据内容使用"我"或"我们"
- 保留真人停顿感，不能太完美、太平滑

**过渡表达**：
- "什么意思呢？"
- "我把它总结为一个公式：..."
- "就用...去类比下"
- "我举个例子"

### Step 4：撰写结尾

选择以下方式之一：
- **开放式问题**："你有没有类似的经历？"、"你会怎么选？"
- **行动建议**："如果觉得有用，可以点赞收藏一下"
- **引导互动**："如果大家感兴趣的话，可以在评论区留言"

**常用收尾**：
- "这样一来...再也不用担心..."
- "如果大家感兴趣的话..."
- "建议你也可以去试试这个方法"
- "好了今天就分享到这里"

## 撰写检查清单

| 检查项 | 合格标准 |
|-------|---------|
| **🚨 2秒退出率（最优先）** | 开头无"前情提要"；教程类建议前2秒先放操作成果画面 |
| 语气风格 | 与样本文稿一致，像朋友聊天 |
| 开头情绪 | 用"我刚发现/忍不住"等真实表达，或直接抛出痛点 |
| 3个问题 | 回应"跟我有什么关系/为什么听你的/能得到什么" |
| 禁止词汇 | 无"谁懂啊/救命/绝了/太牛了"等 |
| 滑梯理论 | 环环相扣，层层递进 |
| 信息密度 | 每句话都有存在理由，无废话 |
| 言之有物 | 能产生行为或认知改变 |
| 句式风格 | 短句为主，用"其实/呢/吧"等语气词 |
| 结尾互动 | 有开放式问题/行动建议/引导互动 |
| AI腔检查 | 不用"作为AI模型"等解释性文字 |
| 真实感 | 保留真人停顿感，不套路不夸张 |
| 人称一致 | "我"或"我们"使用统一 |

---

## 长度控制

**核心原则**：该长则长，该短则短，一切服务于内容本身。

- 不硬性限制字数或时长
- 简单的工具推荐不要注水拉长
- 复杂的深度拆解不要为了控制篇幅而砍掉关键信息
- 唯一的硬约束：**没有废话**

---

## 输出格式

### 默认输出：纯文稿

直接输出完整文案，不要任何说明、分析、标注。

### 可选输出：文稿 + 分镜

如用户说"加分镜"或"需要分镜"，则在每段后添加：

```
【画面提示】此处展示 XXX 截图/演示 XXX 操作
```

---

## 撰写完成后

1. 自检：对照上面的检查清单逐项检查
2. 输出：输出完整文稿
3. 等待用户反馈，或进入 Method 05（评分）
