# Method 01 — 选题流程

## 适用场景

- 只有一个选题方向，没有具体素材
- 想写某个话题但不知道从哪里切入
- 需要搜索最新信息来支撑选题

---

## 执行流程

### Step 1：明确选题核心

回答以下问题：
1. 这个话题想解决什么问题？
2. 目标受众是谁？（参考 user-personas.md）
3. 能覆盖≥2类用户吗？

### Step 2：搜索信息

调用搜索能力，收集以下信息：

| 搜索方向 | 搜索关键词示例 | 目的 |
|---------|---------------|------|
| 核心概念 | "XXX是什么"、"XXX介绍" | 理解基础概念 |
| 最新动态 | "XXX 2024"、"XXX最新" | 获取时效性信息 |
| 使用教程 | "XXX教程"、"XXX怎么用" | 了解具体用法 |
| 用户评价 | "XXX评测"、"XXX体验" | 收集真实反馈 |
| 竞品对比 | "XXX vs YYY" | 找到差异化角度 |

### Step 3：判断 4A 类型

根据选题性质，自动判断内容类型：

| 关键词 | 类型 | 结构 |
|-------|------|------|
| 教程/安装/使用/步骤/方法 | **行动型** | 价值承诺→工具准备→分步操作→总结建议 |
| 分析/解读/趋势/为什么/真相 | **分析型** | 现象引入→拆解分析→底层逻辑→启发总结 |
| 经历/故事/感悟/成长/教训 | **启发型** | 故事开场→转折冲突→反思感悟→行动建议 |
| 心理/行为/现象/观察/人性 | **人类学型** | 现象描述→原因探究→新视角→思考留白 |

### Step 4：生成选题角度

基于搜索结果，提炼 2-3 个可选选题角度：

**角度模板**：
```
角度X：【核心卖点】+【目标人群】+【具体场景】

示例：
角度1：用Claude Skill实现自动化写作（工具教程，覆盖新手+效率追求者）
角度2：Claude Skill vs 传统提示词，效率提升10倍（对比分析，覆盖效率追求者+技术爱好者）
角度3：我用Claude Skill写了100篇文案的经验（实战分享，覆盖效率追求者）
```

### Step 5：用户确认

向用户展示选题选项，等待用户选择：

```
基于你的选题【XXX】，我找到了以下信息：
[简要总结搜索到的核心信息]

建议的选题角度：
1. 【角度1】—— 适合【用户类型】，侧重【价值点】
2. 【角度2】—— 适合【用户类型】，侧重【价值点】  
3. 【角度3】—— 适合【用户类型】，侧重【价值点】

请选择一个角度，我将为你生成大纲。
```

## 选题质量检查

| 检查项 | 合格标准 | 权重 |
|-------|---------|------|
| 三维匹配 | 人×情境×信息定义清晰 | 25% |
| 市场洼地 | 同类内容较少，有差异化 | 25% |
| 信息增量 | 有≥70%的新信息或新视角 | 25% |
| 用户覆盖 | 能覆盖≥2类目标用户 | 25% |

**评分标准**：
- ≥80分：优质选题，可以 proceed
- 60-79分：普通选题，建议优化角度
- <60分：选题较弱，建议换方向

---

## 常见选题陷阱

| 陷阱 | 表现 | 解决方案 |
|-----|------|---------|
| 太硬核 | 纯技术教程（如安装/部署），点击率极低 | 最好包装成具体的场景或收益（如"省下3小时的AI助理"） |
| 太宽泛 | "AI工具推荐" | 缩小到具体场景："设计师必备的3个AI工具" |
| 太专业 | 全是术语，新手听不懂 | 增加解释，降低门槛 |
| 没痛点 | 用户觉得"跟我没关系" | 开头点出具体痛点 |
| 没增量 | 网上已经有很多类似内容 | 找到独特角度或个人经验 |
| 覆盖单一 | 只服务一类用户 | 增加普适性价值点 |
