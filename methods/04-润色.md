# Method 04 — 润色优化

## 适用场景

- 用户已有初版文稿
- 需要优化语气风格、提升质量
- 不改变原意的前提下改进

---

## 执行流程

### Step 1：读取语气样本文稿

**必须执行**：读取 `references/tone-samples/` 中的所有样本文稿，深入理解用户的真实语气风格。

### Step 2：分析原稿

阅读用户的初版文稿，分析：
- 核心主题是什么？
- 目标受众是谁？
- 4A 类型是什么？
- 主要问题在哪里？

### Step 3：润色优化

在不改变原意的前提下，进行以下优化：

| 优化项 | 具体操作 |
|-------|---------|
| **语气匹配** | 调整为麦克斯坦的口语化风格 |
| **开头优化** | 增加情绪，3秒抓住注意力 |
| **句式调整** | 更口语化，多用短句和语气词 |
| **信息密度** | 删减废话，提升信息价值 |
| **过渡平滑** | 优化段落间的衔接 |
| **结尾强化** | 增加互动引导 |

### Step 4：对比输出

向用户展示优化前后的对比：

```
【原文】
XXX

【优化后】
XXX

【优化说明】
- 开头：增加了情绪表达，更抓人
- 句式：调整为更口语化的表达
- 删减：删除了冗余内容，提升信息密度
```

---

## 润色检查清单

| 检查项 | 合格标准 |
|-------|---------|
| 原意保留 | 不改变原文核心观点和关键信息 |
| 语气匹配 | 与样本文稿的口语化风格一致 |
| 开头优化 | 有情绪，3秒抓住注意力 |
| 句式优化 | 短句为主，用"其实/呢/吧"等语气词 |
| 信息密度 | 删减废话，每句话都有价值 |
| 过渡平滑 | 段落间衔接自然 |
| 结尾互动 | 有开放式问题/行动建议 |
| 禁止词汇 | 已删除"谁懂啊/救命/绝了"等 |

---

## 常见润色问题

| 问题类型 | 表现 | 解决方案 |
|---------|------|---------|
| 太书面 | 长句多，像作文 | 拆分为短句，加语气词 |
| 无情绪 | 纯陈述，平淡 | 加"惊到我了/忍不住"等表达 |
| AI腔 | "值得注意的是/综上所述" | 换成口语化表达 |
| 太完美 | 太流畅，像机器写的 | 保留停顿感，加"其实/呢" |
| 开头弱 | 不能3秒抓住人 | 用痛点/反常识/情绪开场 |
| 结尾硬 | 突然结束，无互动 | 加开放式问题或行动建议 |

---

## 输出格式

### 方式一：直接输出优化稿

直接输出润色后的完整文案。

### 方式二：对比输出（推荐）

展示优化前后的对比，让用户看到具体改进。

---

## 润色完成后

1. 自检：对照检查清单逐项检查
2. 输出：输出优化后的完整文稿
3. 等待用户反馈，或进入 Method 05（评分）
